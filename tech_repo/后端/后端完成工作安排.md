## Day 1 — Strapi 管理员账号 + API Token（约 30 分钟）

### 第 1 步：确认 Docker 容器在运行

打开终端，输入：

```bash
docker ps
```

你应该看到两行，分别含 `cssa_postgres` 和 `cssa_redis`，状态为 `Up`。

**如果什么都没有**，说明容器没启动，输入：

```bash
cd /Users/austin/项目/CSSA-Web/backend
docker-compose up -d
```

等几秒，再运行 `docker ps` 确认两个都是 `Up` 状态。

---

### 第 2 步：启动 Strapi

```bash
cd /Users/austin/项目/CSSA-Web/backend/strapi
npm run develop
```

等待约 **30 秒**，终端出现以下字样说明成功：

```
Server listening on: http://localhost:1337
```

> `npm run develop` 会先编译 TypeScript 再启动，每次修改代码后用它。  
> 纯重启（代码没变）也用 `npm run develop`，不要用 `npm start`（start 不编译）。

---

### 第 3 步：打开 Strapi 后台，创建管理员账号

1. 打开浏览器，访问：**`http://localhost:1337/admin`**
2. 第一次进入会显示注册页面，填写：
   - **First name**：`Austin`
   - **Last name**：`Chen`
   - **Email**：你的邮箱（例如 `guangchengchen45@gmail.com`）
   - **Password**：设一个密码（记住，之后登录用）
3. 点击 **Let's start** 按钮

> 如果看到的是登录页面（不是注册），说明之前已有账号，直接登录即可。

---

### 第 4 步：生成 Strapi API Token

登录后台后：

1. 点击左侧菜单最下方的 **⚙️ Settings**（齿轮图标）
2. 在 Settings 左侧列表，点击 **API Tokens**
3. 右上角点击蓝色按钮 **Create new API Token**
4. 填写：
   - **Name**：`microservice-token`
   - **Description**：`NestJS 微服务内部调用`
   - **Token duration**：`Unlimited`（无限期）
   - **Token type**：`Full access`（完全权限）
5. 点击右上角 **Save**
6. 页面弹出一长串 token —— **立刻复制，关掉页面就再也看不到了**

> 如果忘记复制：回到 API Tokens 列表，删掉这个 token，重新生成一个。

---

### 第 5 步：把 Token 写入微服务 .env

```bash
cd /Users/austin/项目/CSSA-Web/backend/microservice
cp .env.example .env
```

用编辑器打开 `backend/microservice/.env`，找到这行：

```
STRAPI_API_TOKEN=在这里粘贴你的token
```

改为你刚复制的 token：

```
STRAPI_API_TOKEN=xxxxxxxxxxxxxxxx你的实际token
```

保存文件。

> ⚠️ `.env` 文件绝对不能提交到 git（里面有密钥）。`.gitignore` 已经排除它了。

## Day 2 — 启动微服务 + 测试完整报名流程（约 45 分钟）

### 第 6 步：安装依赖并启动微服务

**新开一个终端窗口**（不要关 Strapi 那个），运行：

```bash
cd /Users/austin/项目/CSSA-Web/backend/microservice
npm install
npm run start:dev
```

等待约 **15 秒**，看到以下字样说明成功：

```
微服务启动成功，端口：3002
```

**常见报错及解决方法：**

| 报错信息 | 原因 | 解决方法 |
|----------|------|----------|
| `connect ECONNREFUSED 127.0.0.1:6379` | Redis 没启动 | `docker-compose up -d` |
| `connect ECONNREFUSED 127.0.0.1:5432` | PostgreSQL 没启动 | `docker-compose up -d` |
| `Unauthorized` 或 403 | API Token 错误或过期 | 重新生成 token，更新 `.env` |
| `Cannot find module` | 依赖未安装 | `npm install` |

---

### 第 7 步：验证微服务与 Strapi 连通

打开第三个终端窗口，运行：

```bash
curl -s http://localhost:3002/api/v1/registrations \
  -X POST \
  -H "Content-Type: application/json" \
  -d '{"eventId":"999","userInfo":{"name":"连通测试"}}'
```

**正常结果**（返回 503 或 400 都说明通信正常）：

```json
{"message":"无法连接到 CMS 服务，请稍后再试","statusCode":503}
```

**异常结果**（说明微服务没起来）：

```
curl: (7) Failed to connect to localhost port 3002
```

→ 回到第 6 步。

---

### 第 8 步：在 Strapi 后台创建测试活动

1. 打开 `http://localhost:1337/admin`
2. 左侧点击 **Content Manager**
3. 点击 **活动 Event**
4. 右上角 **Create new entry**
5. 填写：
   - **Title**：`测试活动`
   - **Start time**：任意未来时间
   - **Capacity**：`100`
   - **Status**：`active`
   - **Form schema**（粘贴以下 JSON）：

   ```json
   [
     {"field":"name","label":"姓名","type":"text","required":true},
     {"field":"email","label":"邮箱","type":"email","required":true}
   ]
   ```

6. 点击右上角 **Save**，然后点击 **Publish**
7. 记住页面 URL 里的活动 ID（URL 末尾的数字，例如 `.../1`，ID 就是 `1`）
