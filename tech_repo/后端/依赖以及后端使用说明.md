# 依赖以及后端使用说明

> 本文档面向所有需要在本地跑起后端的开发者。
> 如果你是第一次配置，从头到尾按顺序执行即可。

---

## 一、系统依赖

| 依赖 | 版本要求 | 用途 |
|------|----------|------|
| Docker Desktop | 最新版 | 运行 PostgreSQL 和 Redis 容器 |
| Node.js | **20.x**（必须） | 运行 Strapi 和 NestJS |
| nvm-windows | 最新版 | 管理 Node 版本（因为 Strapi 不支持 Node 22+） |
| npm | 随 Node 自带 | 包管理器 |

### 安装 Docker Desktop

1. 打开浏览器，访问 https://www.docker.com/products/docker-desktop/
2. 点击 **「Download for Windows」** 下载安装包
3. 双击安装包，一路 Next，安装完成后**重启电脑**
4. 重启后双击桌面的 Docker Desktop 图标，等左下角状态变成绿色（Running）

### 安装 nvm-windows

打开 **PowerShell（管理员权限）**，执行：

```powershell
winget install CoreyButler.NVMforWindows
```

安装完成后**关闭当前终端，重新打开一个新终端**，然后执行：

```powershell
nvm install 20
nvm use 20
node -v
# 应该输出 v20.x.x，如果不是说明没生效
```

> ⚠️ **每次打开新终端都需要执行 `nvm use 20`**，否则 Node 会回到系统默认版本。
> 如果嫌麻烦，可以把 `nvm use 20` 加到 PowerShell 的启动配置文件里。

## 二、项目结构说明

```
backend/
├── docker-compose.yml    ← 一键启动 PostgreSQL + Redis
├── strapi/               ← CMS 内容管理系统（端口 1337）
│   ├── src/api/          ← 6 个 Content Type（部门/成员/活动等）
│   ├── src/extensions/   ← GraphQL 自定义扩展
│   ├── config/           ← 数据库、插件、中间件配置
│   └── package.json
└── microservice/         ← NestJS 业务微服务（端口 3002）
    ├── src/registration/ ← 报名接口 + Rate Limit
    ├── src/queue/        ← BullMQ 邮件/导出队列
    └── package.json
```

## 三、首次初始化（只做一次）

### 1. 安装 Strapi 依赖

```bash
cd backend/strapi
nvm use 20
npm install
```

等 npm install 完成（可能需要 2-3 分钟，Strapi 依赖很多）。

### 2. 创建 Strapi 环境变量文件

在 `backend/strapi/` 目录下，新建一个名为 `.env` 的文件（注意有个点），填入以下内容：

```env
HOST=0.0.0.0
PORT=1337

# Strapi 安全密钥（实际值从微信群获取，不要自己乱填）
APP_KEYS=your-key-1,your-key-2
API_TOKEN_SALT=your-api-token-salt
ADMIN_JWT_SECRET=your-admin-jwt-secret
TRANSFER_TOKEN_SALT=your-transfer-token-salt
JWT_SECRET=your-jwt-secret

# 数据库（连接 Docker 容器里的 PostgreSQL）
DATABASE_CLIENT=postgres
DATABASE_HOST=localhost
DATABASE_PORT=5432
DATABASE_NAME=cssa_web_dev
DATABASE_USERNAME=postgres
DATABASE_PASSWORD=postgres
DATABASE_SSL=false
```

> ⚠️ 实际密钥值在微信群里，请找 Mike 或 Austin 要。**千万不要把真实密钥提交到 git。**

> ⚠️ `.env` 文件不会被 git 追踪（已在 .gitignore 里），每个人本地都要自己创建。

### 3. 安装 NestJS 微服务依赖

```bash
cd backend/microservice
npm install
```

### 4. 创建微服务环境变量文件

在 `backend/microservice/` 目录下新建 `.env` 文件：

```env
PORT=3002
NODE_ENV=development

# 填入 Strapi API Token（启动 Strapi 后手动获取，见下方说明）
STRAPI_URL=http://localhost:1337
STRAPI_API_TOKEN=

# Redis 连接
REDIS_HOST=localhost
REDIS_PORT=6379

# 前端 CORS
CLIENT_URL=http://localhost:3000
ADMIN_FRONTEND_URL=http://localhost:3001
```

`STRAPI_API_TOKEN` 暂时留空，等 Strapi 启动后再填（见第四节）。

## 四、日常启动流程

**每次开发，按顺序开三个终端：**

### 终端 1 — 启动数据库

```bash
cd backend
docker-compose up -d
```

验证两个容器都正常：
```bash
docker-compose ps
```

输出应该类似：
```
NAME            STATUS
cssa_postgres   Up (healthy)
cssa_redis      Up (healthy)
```

如果 STATUS 不是 `healthy`，等 10-20 秒再看，容器需要时间初始化。

### 终端 2 — 启动 Strapi

```bash
nvm use 20
cd backend/strapi
npm run develop
```

等终端出现这行才算启动成功：
```
info  Server started in XXXX ms
info  http://localhost:1337
```

然后打开浏览器访问：`http://localhost:1337/admin`

**第一次启动**需要注册管理员账号，完成后需要做以下配置（详见 `Austin后续任务SOP.md`）：
- 从 Admin 获取 API Token 并填入微服务 `.env`
- 配置各接口的访问权限
- 录入测试种子数据

### 终端 3 — 启动 NestJS 微服务

> 确保 `STRAPI_API_TOKEN` 已填写后再启动

```bash
cd backend/microservice
npm run start:dev
```

启动成功标志：
```
微服务已启动，端口：3002
```

## 五、端口和服务一览

| 服务 | 地址 | 说明 |
|------|------|------|
| Strapi Admin | http://localhost:1337/admin | 内容管理后台，录入数据用这里 |
| Strapi REST API | http://localhost:1337/api | 所有内容类型的 REST 接口 |
| Strapi GraphQL | http://localhost:1337/graphql | GraphQL Playground，可直接写查询测试 |
| NestJS 微服务 | http://localhost:3002/api/v1 | 报名提交接口（含速率限制） |
| PostgreSQL | localhost:5432 | 数据库，一般不直接访问 |
| Redis | localhost:6379 | 队列和限流，一般不直接访问 |

## 六、停止服务

```bash
# 停止数据库容器（数据保留在 Docker volume 里，下次 up 还在）
cd backend && docker-compose down

# 停止 Strapi / 微服务：在对应终端按 Ctrl + C
```

彻底清空数据库（谨慎）：
```bash
docker-compose down -v   # -v 会删除 volume，数据全没
```

## 七、常见问题排查

**问题：`npm run develop` 报 `Cannot find module`**

```bash
# 重新安装依赖
cd backend/strapi
npm install
```

**问题：Strapi 启动报数据库连接错误**

```
Error: connect ECONNREFUSED 127.0.0.1:5432
```

原因：Docker 容器没起来。解决：

```bash
cd backend
docker-compose up -d
docker-compose ps   # 确认 STATUS 是 healthy
```

**问题：每次打开终端 Node 版本又变回 24**

每次打开终端都要执行 `nvm use 20`。永久解决方案：

```powershell
# 在 PowerShell profile 里加上自动切换
notepad $PROFILE
# 在文件末尾加一行：nvm use 20
# 保存，重新打开终端
```

**问题：5432 端口被占用**

本机装了 PostgreSQL。打开「服务」（Win+R 输入 `services.msc`），找到 PostgreSQL 相关服务，右键 → 停止。

**问题：微服务启动报 `401 Unauthorized`**

`STRAPI_API_TOKEN` 填错或未填。重新从 Strapi Admin → Settings → API Tokens 获取。

**问题：`docker` 命令找不到**

Docker Desktop 没有正常运行，打开 Docker Desktop 等左下角变绿再试。
