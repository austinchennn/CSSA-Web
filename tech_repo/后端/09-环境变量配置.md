---

## 为什么 .env 不能进 git

`.env` 里有数据库密码、JWT 密钥等敏感信息。一旦提交到公开仓库：
- 任何人都能拿到密钥，伪造 JWT token 绕过登录
- 密钥泄漏后只能全部轮换，成本极高

黄金法则：**密钥只在运行环境存在，从不出现在代码历史里**。

---

## strapi/.env — 字段说明

```bash
# Strapi 需要这 5 个密钥来签发 JWT 和 API Token
APP_KEYS=key1,key2          # 两个 key，逗号分隔，用于会话加密
API_TOKEN_SALT=xxx          # API Token 的哈希盐值
ADMIN_JWT_SECRET=xxx        # Admin 登录 JWT 签名密钥
TRANSFER_TOKEN_SALT=xxx     # 数据迁移 Token 盐值
JWT_SECRET=xxx              # 用户 JWT 签名密钥（Users & Permissions 插件使用）
```

所有密钥由 `crypto.randomBytes(32).toString('base64')` 生成，256 位熵值，符合生产级安全要求。

## microservice/.env — 待填字段

```bash
STRAPI_API_TOKEN=待填入
```

这个 token 需要 **Strapi 启动后** 才能获取，步骤：
1. `cd backend/strapi && npm run develop`
2. 打开 `http://localhost:1337/admin`，完成管理员账户注册
3. 左侧菜单 → **Settings → API Tokens → Create new API Token**
4. Name：`microservice-full-access`，Type：`Full Access`
5. 点 Save，复制生成的 Token
6. 填入 `backend/microservice/.env` 的 `STRAPI_API_TOKEN=`

## 验证 .env 不会被提交

```bash
git status
# .env 文件不应该出现在 Changes not staged 或 Untracked files 里
```

如果出现了，说明 .gitignore 没生效，检查：
1. `.gitignore` 里是否有 `.env` 这一行
2. `.env` 是否已经被 git track 过（用 `git rm --cached .env` 解除追踪）
