## 一、Strapi REST API

Strapi 自动为每个 Content Type 生成标准 REST 接口，格式固定。

### 通用响应格式

```json
// 列表查询（find）
{
  "data": [
    {
      "id": 1,
      "attributes": { "name": "...", "title": "..." },
      "meta": {}
    }
  ],
  "meta": {
    "pagination": { "page": 1, "pageSize": 25, "pageCount": 1, "total": 5 }
  }
}

// 单条查询（findOne）
{
  "data": {
    "id": 1,
    "attributes": { "name": "..." }
  },
  "meta": {}
}
```

### 1.1 全站配置 `/api/site-config`

**Single Type — 全站只有一条记录**

| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| GET | `/api/site-config` | Public | 获取全站配置 |
| PUT | `/api/site-config` | Admin | 更新全站配置 |

```json
  "data": {

      "contact_email": "utmcssa@gmail.com",

      "wechat_id": "UTMCSSA",
      "about_description": "我们是...",
      "join_us_description": "每年秋季招新..."
    }
  }
}
```

### 1.2 部门 `/api/departments`

| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| GET | `/api/departments` | Public | 获取所有部门 |
| GET | `/api/departments/:id` | Public | 获取单个部门 |
| POST | `/api/departments` | Admin | 新增部门 |
| PUT | `/api/departments/:id` | Admin | 修改部门 |
| DELETE | `/api/departments/:id` | Admin | 删除部门 |

**常用查询参数：**
```
GET /api/departments?populate=members       # 同时返回该部门的成员
GET /api/departments?sort=name:asc          # 按名称升序
```

**GET /api/departments?populate=members 响应示例：**
```json
{
  "data": [
    {
      "id": 1,
      "attributes": {
        "name": "主席团",
        "leader_name": "张三",
        "introduction": "负责统筹...",
        "members": {
          "data": [
            { "id": 1, "attributes": { "name": "张三", "title": "主席" } }
          ]
        }
      }
    }
  ]
}
```

**POST /api/departments 请求体：**
```json
{
  "data": {
    "name": "市场部",
    "leader_name": "李四",
    "introduction": "负责市场推广..."
  }
}
```

### 1.3 成员 `/api/members`

| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| GET | `/api/members` | Public | 获取所有成员 |
| GET | `/api/members/:id` | Public | 获取单个成员 |
| POST | `/api/members` | Admin | 新增成员 |
| PUT | `/api/members/:id` | Admin | 修改成员 |
| DELETE | `/api/members/:id` | Admin | 删除成员 |

**常用查询参数：**
```
GET /api/members?populate=photo,department          # 同时返回头像和部门
GET /api/members?sort=order:desc                    # 按排序权重倒序
GET /api/members?filters[department][id][$eq]=1     # 筛选某部门的成员
```

**POST /api/members 请求（multipart/form-data，含图片）：**
```
data[name] = 张三
data[title] = 副主席
data[order] = 10
data[department] = 1       # 部门 ID
files[photo] = <图片文件>
```

### 1.4 活动 `/api/events`

| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| GET | `/api/events` | Public | 获取活动列表 |
| GET | `/api/events/:id` | Public | 获取单个活动（含 form_schema）|
| POST | `/api/events` | Admin | 创建活动 |
| PUT | `/api/events/:id` | Admin | 修改活动（含开关报名：修改 status）|
| DELETE | `/api/events/:id` | Admin | 删除活动 |

**常用查询参数：**
```
GET /api/events?filters[status][$eq]=active     # 仅获取开放报名的活动
GET /api/events?sort=start_time:asc             # 按时间升序
GET /api/events?populate=cover_image            # 返回封面图
```

**POST /api/events 请求体：**
```json
{
  "data": {
    "title": "2026 迎新晚会",
    "start_time": "2026-09-05T19:00:00.000Z",
    "location": "MN1250",
    "description": "<p>欢迎所有新生...</p>",
    "capacity": 100,
    "status": "upcoming",
    "form_schema": [
      { "key": "name",  "label": "姓名", "type": "text",   "required": true },
      { "key": "email", "label": "邮箱", "type": "email",  "required": true },
      { "key": "year",  "label": "年级", "type": "select", "required": true,
        "options": ["大一", "大二", "大三", "大四", "研究生"] }
    ]
  }
}
```

**PUT /api/events/:id 开关报名（只改 status）：**
```json
{
  "data": { "status": "active" }
}
```
status 可选值：`upcoming`（待开始）/ `active`（报名中）/ `closed`（已关闭）

### 1.6 报名记录 `/api/registrations`

| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| GET | `/api/registrations` | **Admin** | 查看所有报名（仅管理员）|
| GET | `/api/registrations/:id` | Admin | 查看单条报名 |
| POST | `/api/registrations` | **Public** | 提交报名（前台用户）— 实际经过 NestJS 微服务代理 |
| PUT | `/api/registrations/:id` | Admin | 审批（修改 status）|
| DELETE | `/api/registrations/:id` | Admin | 删除报名 |

**常用查询参数（管理员后台）：**
```
GET /api/registrations?filters[event][id][$eq]=1    # 筛选某活动的报名
GET /api/registrations?filters[status][$eq]=pending # 筛选待审批
GET /api/registrations?populate=event               # 返回关联活动信息
GET /api/registrations?pagination[pageSize]=100     # 分页（导出时用）
```

**PUT /api/registrations/:id 审批通过：**
```json
{
  "data": { "status": "confirmed" }
}
```
status 可选值：`pending`（待审批）/ `confirmed`（已确认）/ `cancelled`（已取消）

## 二、Strapi GraphQL API

GraphQL 端点：`POST http://localhost:1337/graphql`  
请求头：`Content-Type: application/json`

所有 GraphQL 请求格式：
```json
{
  "query": "query GetSiteConfig { siteConfig { ... } }",
  "variables": {}
}
```

### 2.5 提交报名（Mutation）

```graphql
mutation SubmitRegistration($eventId: ID!, $userInfo: JSON!) {
  createRegistration(
    data: {
      event: $eventId
      user_info: $userInfo
      status: "pending"
    }
  ) {
    data {
      id
      attributes {
        status
      }
    }
  }
}
```

**Variables：**
```json
{
  "eventId": "1",
  "userInfo": { "name": "张三", "email": "zs@test.com", "year": "大一" }
}
```

> ⚠️ 注意：实际报名请求应发送到 NestJS 微服务（Port 3002），不要直接调 Strapi，  
> 否则绕过了 Rate Limit 防护。

## 三、NestJS 微服务 API

基础 URL：`http://localhost:3002/api/v1`

### 3.1 提交报名

```
POST /api/v1/registrations
Content-Type: application/json
```

**请求体：**
```json
{
  "eventId": "abc123",
  "userInfo": {
    "name": "张三",
    "email": "zhangsan@mail.utoronto.ca",
    "year": "大一"
  }
}
```

**成功响应（201）：**
```json
{
  "success": true,
  "id": "registration-id-123"
}
```

**错误响应：**

| HTTP 状态码 | 原因 | 响应 |
|------------|------|------|
| 400 | eventId 为空 | `{ "message": "活动 ID 不能为空" }` |
| 400 | 活动未开放 | `{ "message": "报名通道已关闭" }` |
| 400 | 人数已满 | `{ "message": "报名人数已满" }` |
| 429 | 超过限频 | `{ "message": "提交过于频繁，请 60 秒后再试" }` |
| 503 | Strapi 不可达 | `{ "message": "报名提交失败，请稍后重试" }` |

## 四、认证说明

### 公开接口（无需 Token）

- GET `/api/events?filters[status][$eq]=active`
- GET `/api/departments?populate=members`
- GET `/api/members`
- GET `/api/past-events`
- GET `/api/site-config`
- POST `/api/v1/registrations`（NestJS 微服务）

### 需要管理员 Token 的接口

所有增删改操作和查看报名记录都需要 Strapi Admin Token。  
从 Strapi Admin UI 生成：`Settings → API Tokens → Create new API Token`

请求时在 Header 中携带：
```
Authorization: Bearer <your-strapi-api-token>
```

## 五、本地调试快速参考

```bash
# 启动 Strapi
cd backend/strapi && npm run develop
# 访问 http://localhost:1337/admin

# 启动 NestJS 微服务
```
