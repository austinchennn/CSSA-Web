
## 一、先搞懂 3 个核心名词（新手必须看懂）

### 1. 什么是 `main` 分支？

**`main` = 最终上线的正式网站版本**

- **相当于：** 成品仓库
    
- 任何人绝对不能直接在这里改东西
    
- 这里永远放最稳定、可以对外展示的网站
    

> **简单记住：**
> 
> ☑ `main` = 正式上线、不能乱动

### 2. 什么是 `dev` 分支？

**`dev` = 团队所有人开发汇总的地方**

- **相当于：** 半成品组装车间
    
- 所有人做好的功能，先全部放到 `dev` 里面合在一起测试
    
- `dev` 测完没问题，才会放到 `main` 上线
    

> **简单记住：**
> 
> ☑ `dev` = 大家一起开发、合并测试的地方

### 3. 什么是 `feature` 个人开发分支？

**`feature` = 你自己一个人干活的专属小房间**

- 每个人必须自己建一个自己的 `feature` 分支
    
- 你写首页、你写登录、你改样式，只能在这里写
    
- 不能在 `main`、不能在 `dev` 直接写
    

**`feature` 分支名字固定格式（所有人必须按这个起名）：**

- **格式：** `feature-你的名字-你负责的功能`
    
- **例子：**
    
    - `feature-jack-首页制作`
        
    - `feature-lisa-登录页面`
        
    - `feature-tom-网站样式美化`
        

> **简单记住：**
> 
> ☑ `feature` = 你自己专属干活的地方

## 二、所有人电脑统一文件夹位置（Windows + Mac 都适配）

**统一规定：一辈子只放这里，永远不换位置、不删、不移动**

- **Windows 电脑位置：** `C:\Users\你自己的电脑用户名\项目\CSSA-Web`
    
- **Mac 电脑位置：** `用户目录/项目/CSSA-Web`
    

**文件夹名字严格固定：**

- 第一层文件夹：`项目`
    
- 第二层文件夹：`CSSA-Web` （大写 CSSA + 横杠 + 大写 W、小写 eb）
    

> **严格要求：**
> 
> ☑ 所有人目录长得一模一样
> 
> ☑ 不许改名字
> 
> ☑ 不许放桌面
> 
> ☑ 不许放 D 盘乱七八糟位置

## 三、第一步：新手第一次安装软件（只做一次）

只装一个软件：**GitHub Desktop**（纯鼠标图形化，最简单）

**作用：**

- 不用打任何命令
    
- 新建分支、提交、推送、更新全部鼠标点
    
- 安装完登录自己的 GitHub 账号即可

## 四、第二步：新手第一次下载项目到电脑（一辈子只做一次）

1. 在电脑新建第一个文件夹：`项目`
    
2. 打开 GitHub Desktop
    
3. 点：**克隆仓库 (Clone repository)**
    
4. 选择路径放到：`项目 → CSSA-Web`
    
5. 等待下载完成
    

> ☑ 从此以后，你永远只打开这个文件夹写代码

## 五、最重要教学：新手怎么【创建自己的个人 feature 开发分支】

（所有人必须先做这一步，才能开始写代码）

**手把手鼠标点击步骤：**

1. 打开 GitHub Desktop
    
2. 顶部点：**分支 (Branch) → 新建分支 (New branch)**
    
3. 分支名字按格式输入：`feature-你的名字-你的功能`
    
4. **从 `dev` 分支创建新分支（重点！必须选 `dev`）**
    
5. 点创建
    

> ☑ 你现在拥有自己专属干活的房间了
> 
> ☑ 不要在 `main` 干活
> 
> ☑ 不要在 `dev` 干活
> 
> ☑ 你以后写代码，永远只在自己的 `feature` 分支里写

## 六、每天上班开发固定标准流程（新手每天照做）

### 步骤 1：开工第一件事【更新别人最新代码】（必须认真看！）

**作用：** 队友做完的工作都会合并到 `dev` 分支。每天开工前，你必须把 `dev` 里的最新进度拉取到你的 `feature` 分支里，保证你的代码不过时，防止冲突。

**详细鼠标操作步骤：**

1. **获取最新线上代码：** 看向软件正上方偏右，点击 **Fetch origin (获取来源)**。如果按钮变成了 **Pull origin (拉取来源)** 并且带个数字，点击它，等待进度条走完。
    
2. **把 `dev` 的代码合并到你的分支：**
    
    - 点击顶部的 **Current Branch (当前分支)**。
        
    - 在下拉菜单的最底部，找到并点击 **Choose a branch to merge into feature-xxx (选择一个分支合并到...)**。
        
    - 在弹出的列表里，选中 **`dev`** 分支。
        
    - 点击底部的蓝色大按钮 **Create a merge commit (创建合并提交)**。
        

_(恭喜！你现在已经同步了全团队的最新代码！)_

### 步骤 2：确认自己在【自己的 feature 个人分支】

正式敲代码前，左上角看一眼名字，确认是 `feature-xxx` 再开始写代码。

### 步骤 3：打开电脑文件夹写网站代码

打开 `项目 → CSSA-Web` 文件夹。正常改网页、改文字、改样式、做页面。

### 步骤 4：写一点点就提交（新手最重要！）

**什么时候必须提交？**

- 写完一个小按钮
    
- 写完一个板块
    
- 改完一页样式
    
- 修好一个小 bug
    

_(不要一次性写一大堆再提交！写一点、存一点、提交一点)_

**新手怎么点提交（鼠标两步）：**

1. 左下角的框框里写一句话：你刚刚改了什么（比如：“增加了首页的登录按钮”）。
    
2. 点：**Commit to feature-xxx (提交到自己的分支)**。
    

### 步骤 5：点推送（上传到 GitHub 云端）

点软件右上角的蓝字：**Push origin (推送来源)**。这步等于把你的代码备份到网盘。

### 步骤 6：去 GitHub 网页点【新建 Pull Request 申请合并】

**大白话意思：** 我做完了，申请管理员审核，把我的 `feature` 合并到开发仓库 `dev` 里。

### 步骤 7：等管理员审核通过，自动合并

你不用管，等负责人同意就行。

## 七、新手必须分清：代码写到哪里、合并到哪里

1. 你写代码写在自己 `feature` 分支
    
2. 做完申请合并 → 合并到 `dev` 分支
    
3. 全部网站做好测试完 → `dev` 合并到 `main` 正式上线
    

> **永远记住顺序：**
> 
> `feature` → `dev` → `main`

---

## 八、新手绝对不能做的 5 件事（一做就乱）

- ❌ 不许在 `main` 分支写代码
    
- ❌ 不许在 `dev` 分支直接写代码
    
- ❌ 不许删项目文件夹
    
- ❌ 不许重复下载仓库
    
- ❌ 不许自己审核自己的代码
    

---

## 九、新手一句话终极口诀

**固定文件夹不动，自己分支写代码，写一点提交一点，提交申请等人审，做完合并到 dev，最后上线到 main。**
