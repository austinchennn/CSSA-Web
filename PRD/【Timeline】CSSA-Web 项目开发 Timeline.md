## 第一周 4.27 - 5.3｜启动 + 客户端静态页面

### 目标
项目正式启动，完成所有客户端页面的静态 UI（不接真实数据，用假数据占位）。

### 任务清单

**4.27 周日 — 启动会议**
- [ ] 介绍项目架构、技术栈、分支规范
- [ ] 分配模块负责人（客户端页面 / 后台前端 / 后台后端）
- [ ] 所有人创建自己的 feature 分支，确认本地环境跑通
- [ ] 确认设计稿 / Design Token（颜色、字体、间距）

**4.28 - 5.1 — 客户端页面开发（各人并行）**

| 页面 | 包含内容 | 状态 |
|------|---------|------|
| 首页 `Home` | Hero 区域、社团简介、服务说明、精选活动预览、赞助商 Logo 墙、CTA 按钮 | |
| 关于我们 `About` | 组织定位、使命宣言、价值观、服务对象说明 | |
| 管理层 `Team` | 按部门分组的成员卡片（头像、姓名、职位），假数据占位 | |
| 活动 `Events` | 精选活动卡片 + 竖向 Timeline，Framer Motion 滚动动效 | |
| 赞助合作 `Sponsors` | 赞助价值说明、合作案例、档位介绍、Logo 展示区 | |
| 联系我们 `Contact` | 邮箱、社交媒体链接、联系表单（UI 仅，不接后端） | |
| 加入我们 `Join Us` | 招新说明、动态报名表单 UI（假 schema 驱动） | |

**技术要点：**
- 所有页面必须 Desktop + Mobile 响应式适配
- 严格遵守 Design Token（主色 `#BA1C21`，背景 `#FAF8F5` 等）
- 禁止硬编码文案，所有文字用变量/props 传入，为后续接 CMS 做准备

- [ ] 修复 Review 反馈，合并进 `dev`
- [ ] 本周 Demo：浏览器里过一遍所有页面，确认 UI 还原度

### 本周交付物
- ✅ 7 个客户端页面完成静态 UI
- ✅ 全部响应式适配
- ✅ 代码合并进 `dev`

## 第二周 5.4 - 5.10｜数据库设计 + 后端搭建 + 后台前端启动

### 目标
数据库完成建表，Strapi CMS 跑通，后台前端页面开始开发。

### 任务清单

**5.4 周一 — 数据库设计评审会**
- [ ] 确认 6 张数据表的字段细节（见下表）
- [ ] 确认 Strapi Content Type 和表的对应关系
- [ ] 确认 API 接口规范（GraphQL query 和 REST 端点）

**数据库 6 张核心表：**

| 表名 | 关键字段 | 用途 |
|------|---------|------|
| `Site_Configs` | hero_title, hero_subtitle, contact_email, social_links | 全站可配置文案 |
| `Departments` | name, description, display_order | 部门信息 |
| `Members` | name, title, photo, bio, department(FK), display_order | 管理层成员 |
| `Events` | title, date, location, description, cover_image, form_schema, is_active | 活动 + 报名表单 |
| `Past_Events` | title, date, photos, description | 往期活动回顾 |
| `Registrations` | event(FK), user_info(JSON), status, created_at | 报名记录 |

**5.5 - 5.7 — 后端搭建**
- [ ] Docker Compose 启动 PostgreSQL + Redis，确认本地连接正常
- [ ] Strapi 初始化，创建全部 6 个 Content Type
- [ ] 配置 Strapi GraphQL 插件，测试基础 query
- [ ] NestJS 微服务初始化，配置路由基础结构
- [ ] BullMQ + Redis 队列连接验证

**5.5 - 5.8 — 后台前端页面开发**

| 页面 | 包含功能 | 状态 |
|------|---------|------|
| 登录页 `/login` | 邮箱+密码登录，Zod 校验，错误提示 | |
| 仪表盘 `/` | 成员数/活动数/报名数统计卡片，快捷入口 | |
| 成员管理 `/members` | 成员列表、新增/编辑/删除、部门筛选、排序 | |
| 部门管理 `/departments` | 部门列表、新增/编辑/删除、成员数统计 | |
| 活动管理 `/events` | 活动列表、新增/编辑/下线、动态表单 Schema 构造器 | |
| 报名详情 `/events/[id]/registrations` | 报名列表、状态审核、CSV 导出 | |
| 往期活动 `/past-events` | 图片画廊、批量上传、编辑 | |
| 赞助商管理 `/sponsors` | 赞助商 Logo 管理、档位配置 | |
| 全站配置 `/config` | Hero 文案、联系信息、社交媒体链接编辑 | |

**5.9 - 5.10 — PR + 收尾**
- [ ] 后端代码提交 PR，Review 后合并 `dev`
- [ ] 后台前端页面（能完成的部分）提交 PR
- [ ] 本周 Demo：Strapi 后台能增删改数据，GraphQL playground 能查到数据

### 本周交付物

## 风险提示

| 风险 | 概率 | 应对 |
|------|------|------|
| 联调阶段出现大量接口不匹配 | 中 | 第一周就确定好 GraphQL schema，提前对齐 |
| 某成员进度落后影响整体 | 中 | 每周 Demo 强制检查进度，及时 replan |
| 上线前发现严重 Bug | 低 | 第四周预留 3 天专门修 Bug，不新增功能 |
| 域名/部署配置耗时超预期 | 中 | 5.25 开始上线准备，留足 2 天缓冲 |
